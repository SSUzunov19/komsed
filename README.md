# 🧸 КОМСЕД

> Уеб приложение за достъп до релационна база от данни на онлайн магазин за детски играчки и бебешки аксесоари.

Курсов проект по **Информационни системи** — Flask приложение върху **IBM DB2**, което реализира пълен CRUD достъп до базата, обобщена статистика и интерактивна SQL конзола.

## Възможности

- 📊 **Начален панел** — обобщена статистика (брой продукти, категории, клиенти, поръчки)
- 📦 **Продукти** — преглед на каталога, добавяне и изтриване
- 🗂️ **Категории** — йерархия (родител/подкатегория) и брой продукти
- 👤 **Клиенти** — списък с брой поръчки, добавяне
- 🛒 **Поръчки** — списък и детайл с артикулите към всяка поръчка
- 🏷️ **Промоции** — активни и изтекли промоции
- 💻 **SQL конзола** — изпълнение на произволни `SELECT` заявки директно срещу базата


## Модел на данните

Базата съдържа 8 таблици:

| Таблица | Описание |
|---|---|
| `CATEGORY` | Категории с йерархия (саморефериращ се `PARENT_ID`) |
| `PRODUCT` | Продукти с цена, промо цена и наличност |
| `CUSTOMER` | Клиенти |
| `ORDERS` | Поръчки (статус, плащане, доставка) |
| `ORDER_ITEM` | Артикули в поръчка |
| `PROMOTION` | Промоции с период и процент отстъпка |
| `PRODUCT_CATEGORY` | M:N връзка продукт ↔ категория |
| `PRODUCT_PROMOTION` | M:N връзка продукт ↔ промоция |

Дефинирани са първични и външни ключове, уникални ограничения и `CHECK` ограничения за валидност на данните (цени, статуси, периоди и др.).

## Инсталиране и стартиране

### Предпоставки
- Python 3.10+
- Достъп до сървър с IBM DB2
- IBM DB2 клиентски драйвери (инсталират се автоматично с `ibm_db`)

### Стъпки

```bash
# 1. Клониране
git clone <repo-url>
cd Komset

# 2. Виртуална среда (по желание)
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 3. Зависимости
pip install -r requirements.txt

# 4. Конфигурация
cp .env.example .env
# редактирайте .env с реалните данни за връзка с DB2

# 5. Създаване на схемата и зареждане на данните
#    (изпълнете SQL скриптовете през db2 CLI или друг клиент)
db2 -tf sql/create.sql
db2 -tf sql/data.sql

# 6. Стартиране
python app.py
```

Приложението е достъпно на <http://localhost:5000>.

## Конфигурация

Всички данни за връзка се четат от environment променливи (виж `.env.example`). Файлът `.env` **не** се качва в Git.

| Променлива | Описание | По подразбиране |
|---|---|---|
| `DB2_DATABASE` | Име на базата | `SAMPLE` |
| `DB2_HOSTNAME` | Хост на DB2 сървъра | `localhost` |
| `DB2_PORT` | Порт | `50000` |
| `DB2_UID` | Потребител | `db2inst1` |
| `DB2_PWD` | Парола | _(празно)_ |
| `DB2_SCHEMA` | Схема | `FN2MI0700362` |
| `SECRET_KEY` | Flask secret key | `comsed-dev-secret-key` |
